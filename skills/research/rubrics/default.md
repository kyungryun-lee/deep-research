# 실무 솔루션 평가 기준 (Default — Production Solution)

검증된 솔루션 중 가장 효율적인 것을 선택하기 위한 기준입니다.
Proven(검증됨)과 Actionability(실행가능)의 가중치가 가장 높습니다.

## 가중치 (Evaluator 자동 추출용)

```json
{"core":{"accuracy":0.15,"coverage":0.10,"recency":0.10,"structure":0.05},"context":{"proven":0.20,"actionability":0.20,"efficiency":0.10,"env_fit":0.10},"sea_threshold":75}
```

## 고정 코어 (40%)

### Accuracy (정확성) — 15%
| 점수 | 기준 |
|------|------|
| 90-100 | 모든 수치 교차검증, 할루시네이션 0, 소스 등급 정확 |
| 70-89 | 대부분 정확, 1-2건 미확인 |
| 50-69 | 일부 부정확, arXiv를 S등급으로 과대분류 |
| 0-49 | 다수 오류, 환각 |

### Coverage (범위) — 10%
| 점수 | 기준 |
|------|------|
| 90-100 | 핵심 영역 전부 포괄, 환경 제약까지 포함 |
| 70-89 | 주요 영역 포함, 1-2개 부차 누락 |
| 0-69 | 핵심 영역 누락 |

### Recency (최신성) — 10%
| 점수 | 기준 |
|------|------|
| 90-100 | 최근 1-2년 소스 90%+ |
| 70-89 | 70-89% |
| 0-69 | 50% 미만 |

### Structure (구조) — 5%
| 점수 | 기준 |
|------|------|
| 90-100 | 논리적 흐름, 비교표 활용, 중복 없음 |
| 0-69 | 구조 불명확 |

## 맥락 확장 (60%)

### Proven (검증됨) — 20%
| 점수 | 기준 |
|------|------|
| 90-100 | 핵심 솔루션 전부 L3(프로덕션), 사례/버전/사용기업 명시 |
| 70-89 | L3 다수 + L2 일부 |
| 50-69 | L2 위주, L3 부족 |
| 0-49 | L1/L0 위주, 검증 근거 없음 |

### Actionability (실행가능) — 20%
| 점수 | 기준 |
|------|------|
| 90-100 | 복사-실행 코드, API 시그니처, 설정 파일, 설치 명령 |
| 70-89 | 코드 스니펫 포함, 대부분 직접 활용 가능 |
| 50-69 | 코드 일부, 나머지 설명만 |
| 0-49 | 구현 정보 전무 |

### Efficiency (효율성) — 10%
| 점수 | 기준 |
|------|------|
| 90-100 | 대안 비교 + 순차 적용 모델 + 보수적/낙관적 범위 |
| 70-89 | 정량 비교 있으나 범위 미제시 |
| 0-69 | 정성적 비교만 |

### Environment Fit (환경적합) — 10%
| 점수 | 기준 |
|------|------|
| 90-100 | 환경 제약 분석 + 구체적 통합 방안 + 테스트 방법 |
| 70-89 | 환경 이해 있으나 일부 제약 미반영 |
| 0-69 | 환경 무관한 일반론 |

## 소스 등급 (실무 검증 중심)
- **S**: 공식 문서 + 프로덕션 검증 GitHub (v1.0+, 활발한 유지보수)
- **A**: 기업 엔지니어링 블로그 (실적용 사례) + 코드 동반 학술 논문
- **B**: 개발자 블로그, 커뮤니티, arXiv (코드 미동반)
- **C**: 이론만, 코드 없음, 마케팅

## SEA 충족률 기준
- 75% 이상: 충분

## 캘리브레이션 앵커 (Evaluator 참고용)

### 점수 90 예시 (High Quality)
- 핵심 주장 100% 교차검증 (2+독립 소스), 할루시네이션 0건
- S/A 등급 소스 80%+, 프로덕션 검증 코드/설정 포함
- 최근 1-2년 소스 90%+, 환경 제약 분석 + 구체적 통합 방안
- 복사-실행 가능한 코드, API 시그니처, 설치 명령 포함
- 순차 적용 모델로 실제 절감률 계산 (보수적/낙관적 범위)

### 점수 70 예시 (Acceptable)
- 주요 영역 포함, 1-2건 미확인 주장 존재
- A/B 등급 소스 위주, L2-L3 검증 수준
- 코드 스니펫 일부 포함, 대부분 설명 위주
- 정량 비교 있으나 범위 미제시
- 환경 이해 있으나 일부 제약 미반영

### 점수 50 예시 (Needs Improvement)
- 핵심 영역 일부 누락, B/C 등급 소스 다수
- L1/L0 미검증 솔루션 위주, 구현 정보 부족
- 환경 무관한 일반론, arXiv를 S등급으로 과대분류
- 정성적 비교만, 코드 없음

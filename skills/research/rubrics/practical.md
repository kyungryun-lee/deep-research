# 실무 구현 평가 기준 (Practical Rubric)

실제 구현, 도구 선택, 아키텍처 결정에 특화된 평가 기준입니다.
Actionability의 가중치가 높고, 코드 예시와 구현 가이드를 중시합니다.

## 가중치 (Evaluator 자동 추출용)

```json
{"weights":{"scope":0.10,"literature":0.15,"analysis":0.15,"recency":0.15,"actionability":0.25,"organization":0.05,"references":0.15},"sea_threshold":75}
```

## 차원별 기준

### 1. Scope (범위) — 가중치 10%
범용 rubric과 동일하되, **구현 단계별 커버리지** (설계→구현→테스트→배포)를 추가 평가합니다.

### 2. Literature (문헌 품질) — 가중치 15% (↓)
| 점수 | 기준 |
|------|------|
| 90-100 | 공식 문서(S) + 기업 엔지니어링 블로그(A) 60%+, 실제 프로젝트 사례 포함 |
| 70-89 | 공식 문서 포함, 실무 사례 일부 |
| 50-69 | 공식 문서 부족, 일반 블로그 위주 |
| 30-49 | 검증되지 않은 소스 위주 |
| 0-29 | 소스 거의 없음 |

### 3. Analysis (분석 깊이) — 가중치 15%
범용 rubric과 동일하되, **대안 비교** (A vs B), **트레이드오프 분석**, **환경별 적합성** 평가를 추가합니다.

### 4. Recency (최신성) — 가중치 15%
범용 rubric과 동일. 실무 도구는 빠르게 변하므로 최신성이 중요합니다.

### 5. Actionability (실행가능성) — 가중치 25% (↑↑)
| 점수 | 기준 |
|------|------|
| 90-100 | 복사-실행 가능한 코드, API 시그니처, 설정 파일, 디렉토리 구조, 설치 명령 |
| 70-89 | 코드 스니펫 포함, 대부분 직접 활용 가능 |
| 50-69 | 코드 일부, 나머지는 설명만 |
| 30-49 | 개념적 설명 위주, 코드 거의 없음 |
| 0-29 | 구현 정보 전무 |

### 6. Organization (구조) — 가중치 5%
범용 rubric과 동일하되, **단계별 가이드 형태** (Step 1→2→3)를 선호합니다.

### 7. References (참조 정확성) — 가중치 15%
범용 rubric과 동일하되, **GitHub 저장소 URL** 및 **패키지 버전**의 정확성을 추가 검증합니다.

## SEA 충족률 기준
- Practical rubric에서는 **75% 이상**이 충분 기준 (약간 완화 — 실무는 변화가 빠름)
